## AgentSentry v4.1.0-beta (Historical)

> **Note:** This project has been re-versioned to v0.5.0-beta to honestly reflect its maturity stage. The v4.x version numbers were inherited from an earlier prototype and did not represent years of iteration.

Memory-aware management and safety framework for AI agents. Your agents forget everything between sessions — AgentSentry remembers.

### Highlights

- **28 fixes** from comprehensive SDLC analysis (security, performance, code quality, CI/CD, docs)
- **0 production vulnerabilities** (npm audit clean)
- **1113 tests passing** across unit, contract, e2e, and performance suites
- **Rebranded** from AgentOps → AgentSentry

### Added
- `agent_sentry_generate_handoff` — 10th MCP tool for structured session handoffs
- `errorMessage()` shared utility for consistent error formatting (25+ files)
- SIGTERM graceful shutdown handler
- npm audit step in CI pipeline
- ESLint `no-floating-promises` and `no-misused-promises` rules
- CI concurrency control (cancels stale runs)

### Fixed (Critical & High)
- **P1:** Startup no longer loads entire events table — uses `getLatestHash()` instead
- **S1:** Auth bypass warning when no access key configured
- **S2:** PostgREST filter injection in Supabase provider — inputs sanitized
- **S8:** Query parameter authentication removed; header-only auth enforced
- **S13:** CORS defaults to `localhost` instead of wildcard
- **S14:** UUID validation on all Supabase provider inputs
- **P2–P8:** Query optimizer, busy_timeout, smarter cache invalidation, embedding LRU cache, parallel Supabase queries, transaction-wrapped batch inserts

### Security
- Patched `path-to-regexp` and `picomatch` high-severity dependency vulnerabilities
- ONNX model checksum verification on load

### Quick Start

npm install @calabamatex/agentsentry
npx @calabamatex/agentsentry init
